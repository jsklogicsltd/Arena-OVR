## Arena OVR

Arena OVR is a **Flutter + Firebase** app for high school/team sports programs where **coaches award points** to athletes across four pillars and the app computes an **OVR (0–99)** that reveals over time during a season.

### What the app does

- **Coach experience**
  - Create/manage teams and seasons
  - **Manage Roster**: assign each athlete a **Position Group** (OFFENSE/DEFENSE/SPECIAL TEAMS) and optional **Custom Tags** (e.g., Seniors, JV)
  - **Award Points**: bulk-award points to multiple athletes using four pillars + a “challenge” (subcategory) and an optional note
  - View roster sorted by OVR, team average OVR, and team feed
- **Athlete experience**
  - View personal dashboard, point history, streaks, and OVR (with Day 1 lock + daily cap reveal)
  - Leaderboard + team feed

### Tech stack

### Data model (core concepts)

There are **no separate Athlete/Coach models**. Everyone is stored in Firestore as a **`users`** document and differentiated by `role`:

- `users/{uid}`: `role` is one of `superadmin`, `coach`, `athlete`
  - Athletes typically use `teamId`
  - Coaches typically use `teamIds` + `activeTeamId`
- `teams/{id}`: holds team settings including `positionGroups` + `customTags`
- `seasons/{id}`: season configuration (start date drives the day/cap schedule)
- `transactions/{id}`: point awards ledger entries
- `feed/{id}`: team activity feed items
- `notifications/{id}`: in-app notifications

### Scoring & OVR (how it works)

#### The four pillars

The four parent categories are:

- **Athlete** (weight 0.4) — formerly "Performance"
- **Student** (weight 0.2) — formerly "Classroom"
- **Teammate** (weight 0.2) — formerly "Program"
- **Citizen** (weight 0.2) — formerly "Standard"

Coaches award points using a **challenge (subcategory)** for each pillar (see `ChallengeCatalog`). Old Firestore data using the legacy names is handled via backward-compatible aliases in `RatingRepository` and `LeaderboardController`.

#### Source of truth: transactions ledger

Each award creates one Firestore doc in **`transactions`** per:

- \(athlete × category\)

After writes, the app **recalculates OVR** for the whole team/season and stores per-athlete aggregates on the user document:

- `currentRating`: `{ Athlete, Student, Teammate, Citizen }`
- `ovr` (displayed/capped), `actualOvr` (true), `ovrDay`, `ovrCap`

#### Dual OVR system and FinalOVR merge

The app has two independent scoring systems that merge into one **FinalOVR**:

| System | Source | Range | Field |
|--------|--------|-------|-------|
| **Manual OVR** | Coach awards points via 4 categories | 0-99 | `actualOvr` |
| **Automated OVR** | Scoring engine converts raw athletic data into Performance Points | 30-99 | `automatedOvr` |

**Merge formula:** `FinalOVR = (ManualOVR + AutomatedOVR) / 2`, clamped to 0-99.
If an athlete has no assessment data, `FinalOVR = ManualOVR` (zero-disruption fallback).

The `UserModel.finalOvr` getter implements this. `coachVisibleOvr` delegates to `finalOvr`, so all leaderboard sorts and coach-facing displays automatically use the merged value.

#### Manual OVR formula and reveal schedule

- Each category total is clamped to **0..100**
- OVR starts from a base of **50**, then adds a weighted sum (scaled by 0.99), and clamps to **0..99**
- **Day 1 is locked** (athletes can’t see OVR)
- Day 2+ OVR is shown but **capped** by a daily schedule (fully unlocks at Day 9)

All math + cap schedule lives in `lib/app/core/services/ovr_engine_service.dart`.

### Automated Assessment Engine (scoring engine)

Files live in `lib/app/scoring_engine/`:

- **`profile_assignment.dart`** — assigns `PowerProfile` (light/medium/heavy) and `SpeedProfile` (standard/heavy) from height + weight.
- **`scoring_engine.dart`** — `scoreEvent`, `calculateNumbers`, `assignOverallRatings` (team-relative OVR).
- **`tier_tables.dart`** — all tier thresholds for 8 events x 4 grades x 2–3 profiles.

Physical profile data (`grade`, `heightInches`, `weightLbs`, `powerProfile`, `speedProfile`) is stored on each `UserModel` document. Players update these values from **Settings > Physical Profile**, which auto-computes and persists the power/speed profiles via `PlayerController.updatePhysicalProfile()`.

The **Award Points > Assessments** tab lets coaches input raw event data. On submit, `CoachController.submitAssessment()` runs the scoring pipeline (`scoreEventByName` -> `calculateNumbers`) and writes `automatedOvr` + `assessmentData` to each athlete's Firestore document.

### Admin features

- **Admin Dashboard** (`admin_dashboard_view.dart`): overview of all schools, coaches, and athletes
- **Create School**: generates an invite code for coaches to join
- **Delete School**: red trash icon on each school card with confirmation dialog. Performs a Firestore batch delete of the school document plus all associated teams, users, seasons, transactions, and feed items.
